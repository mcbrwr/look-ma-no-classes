Look ma! No classes!

An excercise to get a grip on the magic css selectors and to get acquainted with some beautiful html elements that can help you cure diveritus.

This is by not by a stretch an example of best practices

It is however one of the best ways to get to grip on the magic of css selectors. By getting grip on these, you'll apply class names a lot more efficient.

How to use this exercise

Copy the source of this document, strip the <style> block from the <head>, then rebuild by adding your own css. None of the html may be modified, no classes added, all styling must be done by using the untouched and serene html elements. Make a perfect match with this document and you can call yourself a css master.

"This is not really beautiful design is it?"
No it is not. But in a way it is.. It is pure html, styled with magic css. There is beauty in that!
"I don't like dropdown menu's"
Me neither but it is an excellent excercise to build one with only css
"This is not responsive!"
Hold your horses.. Working on it.
"I know something to make this more fancy!"
Me to. Make a pull request and let's make this a great frontend assessment
